Local Storage: This read-only interface property provides access to the Document’s local storage object, the stored data is stored across browser sessions. Similar to sessionStorage, except that localStorage data gets cleared when the page session ends – that is when the page is closed. It is cleared when the last “private” tab of a browser is closed (localStorage data for a document loaded in a private browsing or incognito session).

Session Storage: Session Storage objects can be accessed using the sessionStorage read-only property. The difference between sessionStorage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.

Block Scope: A variable when declared inside the if or switch conditions or inside for or while loops, are accessible within that particular condition or loop. To be consise the variables declared inside the curly braces are called as within block scope.

Global Scope: Variables declared Globally (outside any function) have Global Scope. Global variables can be accessed from anywhere in a JavaScript program. Variables declared with var, let and const are quite similar when declared outside a block.

Event loop is an endless loop, which waits for tasks, executes them and then sleeps until it receives more tasks. JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. The event loop is a mechanism that allows JavaScript to perform non-blocking operations. When an asynchronous task is started, the event loop will start running. Once the task is completed, the event loop will again check for any other tasks that need to be performed.

Undefined means a variable has been declared but has not yet been assigned a value. undefined is a type by itself (undefined).
  • A variable that has not been assigned a value is of type undefined.
  • A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value.
  • A function returns undefined if a value was not returned.
  • Accessing a non-existing property
  • Without return statement, a JavaScript function returns undefined
  • You get undefined when accessing an array element with an out of bounds index.
